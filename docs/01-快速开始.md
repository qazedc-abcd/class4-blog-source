# 01 快速开始

本文用最快速度，让你在本地或服务器上跑起来。

---

## 准备环境

需要安装：

- Docker + Docker Compose（服务器上 1Panel 通常已经装好）
- Git
- Python 3.12+（可选，仅用于生成密码 hash）

---

## 1. 克隆项目

```bash
git clone https://github.com/qazedc-abcd/class4-blog-source.git classmemorial
# 或者如果已经把新框架提交到另一个仓库，请用那个仓库地址。
cd classmemorial
```

> 注意：这里我们以 `classmemorial` 作为新仓库名。如果你要继续用旧仓库，建议先把原 Hexo 代码切到新分支，再把本框架内容 force-push 到 `main`。

---

## 2. 复制环境变量

```bash
cp .env.example .env
```

用任意编辑器打开 `.env` 填写。

---

## 3. 生成管理员密码

```bash
python scripts/hash_password.py
```

按提示输入密码，输出类似：

```
ADMIN_PASSWORD_HASH=pbkdf2_sha256$200000$xxxx$yyyy
```

把这一行贴进 `.env`。

> 建议用 `ADMIN_PASSWORD_HASH` 而不是 `ADMIN_PASSWORD`，更安全。如果你只是本地试试，可以只设 `ADMIN_PASSWORD`。

---

## 4. 填写 R2 参数（图片上传用）

在 Cloudflare 控制台：

1. 进入 **R2** → 你的桶。
2. 点击 **Manage R2 API Tokens** → **Create API Token**。
3. 权限选 **Object Read & Write**。
4. 复制 **Access Key ID** 和 **Secret Access Key**。

填写到 `.env`：

```env
R2_ACCOUNT_ID=你的账户ID
R2_ACCESS_KEY=Access Key ID
R2_SECRET_KEY=Secret Access Key
R2_BUCKET=class4
R2_PUBLIC_BASE=https://drive.zichenccc.cn/raw
```

`R2_PUBLIC_BASE` 是桶的公开访问域名。例如你原来的图片链接是 `https://drive.zichenccc.cn/raw/班级blog/wx.jpg`，那就填 `https://drive.zichenccc.cn/raw`。

---

## 5. 填写 GitHub 参数（备份同步用）

1. 打开 GitHub → Settings → Developer settings → Personal access tokens → **Tokens (classic)**。
2. 生成一个 token，勾选 **`repo`** 权限。
3. 复制 token。

填写到 `.env`：

```env
GITHUB_REPO=qazedc-abcd/class4-blog-source
GITHUB_BRANCH=main
GITHUB_TOKEN=ghp_xxxxxxxx
GITHUB_WEBHOOK_SECRET=随便填一个字符串，Webhook 配置里也要用
SYNC_ENABLED=true
```

---

## 6. 启动

```bash
docker compose up -d --build
```

第一次启动会：

- 构建后端镜像（安装 Python 依赖）
- 构建 nginx 镜像（复制后台页面）
- 初始化 content 目录
- 自动生成一次静态站点

---

## 7. 验证

- 站点首页：`http://<服务器IP>/`
- 后台管理：`http://<服务器IP>/admin/`

用 `.env` 里设置的密码登录后台，试着写一篇文章并保存。

---

## 8. 域名 + EdgeOne（下一步）

## 常用命令

```bash
# 查看日志
docker compose logs -f backend

# 重启
docker compose restart

# 更新代码后重新构建
docker compose up -d --build

# 进入后端容器调试用
docker compose exec backend sh

# 停止
docker compose down
```
