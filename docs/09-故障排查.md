# 09 故障排查

遇到问题先来这里查。

---

## 通用排查步骤

1. 看日志：

   ```bash
   docker compose logs -f backend
   ```

2. 确认容器在跑：

   ```bash
   docker compose ps
   ```

3. 确认 `.env` 已填写完整。

---

## 站点打不开

### 现象：浏览器显示"无法访问此网站"

可能原因：

- 服务器防火墙没放行 80 端口。
- Docker 没启动。
- nginx 容器没启动。

排查：

```bash
docker compose ps
# 应该看到 classmemorial-backend 和 classmemorial-nginx 都是 Up
```

如果 nginx 没起来，看日志：

```bash
docker compose logs nginx
```

---

## 后台打不开或登录失败

### 现象：`/admin/` 空白

- 检查是否带了末尾 `/`：`/admin/` 而不是 `/admin`。
- 如果 nginx 返回 404，可能是构建时没把 `admin/` 复制进去，重新构建：

  ```bash
  docker compose up -d --build
  ```

### 现象：登录时提示"密码错误"

- 检查 `.env` 里的 `ADMIN_PASSWORD_HASH` 或 `ADMIN_PASSWORD`。
- 如果用了 `ADMIN_PASSWORD_HASH`，确保 hash 生成正确。

---

## 保存文章后站点没更新

可能原因：

1. backend 报错了。看日志：

   ```bash
   docker compose logs -f backend
   ```

2. 重建锁卡住了。如果上一次的重建进程还在运行，新的保存会提示"another build in progress"。等 2 分钟或重启 backend：

   ```bash
   docker compose restart backend
   ```

3. EdgeOne 缓存。如果站点已经通了 HTTPS，EdgeOne 可能缓存了旧页面。到 EdgeOne 控制台刷新缓存。

---

## GitHub 同步失败

### 后台改后 GitHub 没提交

看日志：

```bash
docker compose logs -f backend
```

常见错误：

- `GITHUB_TOKEN` 没填或权限不足。
- `GITHUB_REPO` 写错。
- 网络不通 GitHub。

### GitHub 改后服务器没更新

1. 检查 GitHub Webhook 的 Recent Deliveries：
   - 状态码是否 200？
   - 如果 401，说明 `GITHUB_WEBHOOK_SECRET` 不一致。
   - 如果 404，说明域名解析没指向服务器或 EdgeOne 拦截了。

2. 检查 EdgeOne 是否放行 `/api/webhook/github`。

3. 手动测试拉取：

   ```bash
   docker compose exec backend sh
   cd /app/content
   git status
   git log -1
   ```

---

## R2 图片上传失败

看日志：

```bash
docker compose logs -f backend
```

常见错误：

- `R2_ACCESS_KEY` 或 `R2_SECRET_KEY` 填错。
- `R2_BUCKET` 不存在。
- `R2_ACCOUNT_ID` 填错。
- API Token 没有写权限。

---

## 协议弹窗不弹

可能原因：

- 协议弹窗被关闭：`content/site.yml` 里 `agreement.enable` 为 false。
- 已经同意过。清除浏览器 localStorage 中的 `site_agreement_v1` 再刷新。
- 协议页面（`/用户协议/`、`/免责声明/`）打不开。检查这两个页面是否生成成功。

---

## 图库不显示图片

- 检查 `content/gallery/manifest.yml` 是否存在。
- 检查 `content/site.yml` 里 `gallery.enable` 是否为 true。
- 检查图片 URL 是否能直接访问（把 URL 贴到浏览器地址栏）。

---

## 日志提示 "Permission denied"

如果 Docker 卷权限不足，可以手动改权限：

```bash
docker compose down
sudo chown -R 1000:1000 /var/lib/docker/volumes/classmemorial_*
docker compose up -d
```

或者更简单：在 `docker-compose.yml` 里给 backend 加 `user: root`（不推荐长期用）。

---

## 仍有问题

1. 先看日志找错误。
2. 检查 `.env` 每一项是否都填对。
3. 在 GitHub 仓库 Issues 里搜索类似问题。
4. 如果是新的错误，可以查看 `content/` 目录的文件是否完整。
