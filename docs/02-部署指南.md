# 02 部署指南：1Panel + Docker

本文假设你的服务器是 **2H2G**、已经装了 **1Panel**、**Python**、**astrbot**，并且你有一块域名 `class4.zichenccc.cn`。

---

## 目标

把 `classmemorial` 用 Docker 跑起来，
域名指向服务器，EdgeOne 负责加速（下一篇讲）。

---

## 第一步：上传代码到服务器

### 方法 A：Git 拉取（推荐）

在服务器上：

```bash
cd /opt  # 或你喜欢的目录，如 /home/yourname
sudo git clone https://github.com/qazedc-abcd/class4-blog-source.git classmemorial
cd classmemorial
sudo cp .env.example .env
sudo nano .env  # 按 01-快速开始 填写
```

### 方法 B：通过 1Panel 文件管理器

在 1Panel 里创建目录 `/opt/classmemorial`，把本地压缩包上传解压，再在线编辑 `.env`。

---

## 第二步：1Panel 里创建 Compose 容器（可选但推荐）

1. 登录 1Panel → **容器** → **编排**。
2. 点击 **创建编排**。
3. 编排名称填 `classmemorial`。
4. 路径选择 `/opt/classmemorial`（里面要有 `docker-compose.yml`）。
5. 点击 **确定**，1Panel 会自动 `docker compose up`。

如果不喜欢用 1Panel，直接 SSH 运行：

```bash
cd /opt/classmemorial
sudo docker compose up -d --build
```

---

## 第三步：放行端口

`docker-compose.yml` 里 nginx 映射了宿主机的 **80** 端口。

在 1Panel → **防火墙** 里放行 **80 / TCP**。

> 注意：你不需要放行 backend 的 8000 端口，因为外部访问只走 nginx 反代到 backend，不直接暴露。

---

## 第四步：域名解析到服务器

进入你的域名 DNS 管理（腾讯云 DNSPod / Cloudflare）：

- 记录类型：`A`
- 主机记录：`class4`
- 记录值：你的服务器公网 IP
- TTL：10 分钟或 600 秒

等待几分钟生效后，访问 `http://class4.zichenccc.cn/` 应该能看到站点。

---

## 第五步：配置 HTTPS（可选）

你后续会通过 **EdgeOne** 套 SSL，所以服务器本身不需要额外证书。EdgeOne 提供 SSL 证书。

如果你不想用 EdgeOne，可以在 1Panel → **网站** → 反向代理 → 申请 Let's Encrypt 证书。
但既然你明确要用 EdgeOne，建议直接按下一篇配置。

---

## 第六步：确认后台可访问

访问 `http://class4.zichenccc.cn/admin/`。

注意末尾的 `/`，不加可能跳不到后台 SPA。

---

## 目录权限说明

Docker 运行后，会创建三个卷：

| 卷名 | 用途 | 挂载位置 |
|---|---|---|
| `content` | Markdown / site.yml / 图库清单 | 后端 `/app/content` |
| `site` | 生成的静态站 | 后端 `/app/data/site` 和 nginx `/usr/share/nginx/html/site` |
| `dbdata` | 预留数据目录 | 后端 `/app/data` |

这些卷由 Docker 管理，无需手动处理。如果你需要手动备份内容，看 [04-GitHub同步配置](04-GitHub同步配置.md)。

---

## 与 astrbot 共存

2H2G 同时跑 astrbot 和本站点是可以的，但建议：

- 在 1Panel 容器里给 `classmemorial-backend` 设置 **内存限制 256MB**（`docker-compose.yml` 已写）。
- 如果内存吃紧，把 astrbot 也限制到 512MB 以内。
- 不要给 nginx 容器分配太多资源（已限制 64MB）。
