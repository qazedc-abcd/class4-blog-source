# 00 总览：ClassMemorial 是什么

如果你是接手这个站点的维护者，请先读这一篇。本文不讲命令，只讲**整体思路**。

---

## 一句话概括

ClassMemorial = 一个用 Python 写的**班级纪念站框架**。它把博客文章、图库、协议弹窗、评论、备案信息显示、后台管理，全部封装进一个 Docker 项目里，部署在你的服务器上。

原来的网站用 **Hexo + Fluid 主题** 写的，发布到 **Cloudflare Pages**。迁移后：

| 原来的 | 现在的 |
|---|---|
| Hexo 主题 | 自研模板 |
| Cloudflare Pages 自动构建 | 服务器内 Docker + 后端自动构建 |
| Decap CMS + GitHub OAuth | 自研 Web 后台（`/admin`） |
| 图片上传位置分散 | 全部走 Cloudflare R2 |
| 没有图库 | 新增图库功能 |
| 备案信息硬编码 | 后台可开关 |
| 单向推送（GitHub → CF Pages） | 双向同步（GitHub ↔ 服务器） |

---

## 三个核心概念

### 1. 内容 = Markdown 文件

所有文章、单页、配置都存在 `content/` 里：

- `content/posts/*.md` —— 博客文章
- `content/pages/*.md` —— 单页（用户协议 / 免责声明 / 关于）
- `content/site.yml` —— 站点标题、备案、Twikoo 等设置
- `content/gallery/manifest.yml` —— 图库索引

这些文件最终会被提交到 **GitHub 仓库**，既做备份，又做版本控制。

### 2. 一次修改，两段更新

这是你最关心的"联动"。

**后台改 → 服务器 + GitHub 都更新：**

```
你在 /admin 编辑文章
       ↓
backend 把改动写入 content/
       ↓
backend 重建静态站点 → 生成 site/
       ↓
nginx 立即提供最新站点
       ↓
backend 同时 git commit & push 到 GitHub
```

**GitHub 改 → 服务器也更新：**

```
你直接在 GitHub 网页/手机 App 修改 Markdown
       ↓
GitHub 触发 Webhook → POST 到服务器
       ↓
backend 执行 git pull → 拉下最新内容
       ↓
backend 重建静态站点 → 更新 site/
```

所以 GitHub 是**备份仓库**，服务器是**运行现场**，两者始终同步。

### 3. 图片放在 Cloudflare R2

继续使用你现有的 R2 桶。后台写文章时，点击"插入 R2 图片"上传；
图库页面里上传照片也直接进 R2。R2 返回直链，写入 Markdown 或图库清单。

## 资源占用

在 2H2G（2 核 2G）服务器上实测：

- `classmemorial-backend`：约 150–200MB RAM
- `classmemorial-nginx`：约 10–20MB RAM
- **合计 < 250MB**，与 astrbot 一起跑不会爆内存。

## 你需要准备什么

1. 一台服务器（已有 1Panel 更好）。
2. 一个域名（`class4.zichenccc.cn`）。
3. 一个 Cloudflare 账号（R2 + EdgeOne）。
4. 一个 GitHub 账号（公开备份仓库）。
5. 一点点耐心，跟着文档一步步走。
